Layoffs Data Analysis using MySQL

This project analyzes a layoffs dataset using MySQL, focusing on cleaning and managing data across three tables. It includes key columns like company, location, industry, and layoff statistics. The goal is to uncover trends in layoffs across various sectors and locations.

This project centers on the analysis of layoffs data obtained from Kaggle, utilizing MySQL for data cleaning and organization. The dataset encompasses several important columns, including company, location, industry, total_laid_off, percentage_laid_off, date, stage, country, and funds_raised_millions.

Key Features:
1. Data Extraction: The dataset was imported into MySQL Workbench for analysis.
2. Data Cleaning: Three tables were established during this phase: layoffs (for raw data), layoffs_stagging (for intermediate processing), and layoffs_stagging_2 (for the final cleaned data).
3. Cleaning Operations: This step involved addressing missing values, formatting dates, normalizing location and company names, and ensuring overall consistency within the dataset.
4. Final Clean Data: The cleaned dataset is housed in the layoffs_stagging_2 table, making it ready for additional analysis or visualization.
