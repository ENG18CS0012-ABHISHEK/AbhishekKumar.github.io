Heart Attack Analysis & Prediction Using Machine Learning

This project focuses on predictive modeling for heart attack risk using a dataset of 14 key health attributes, including age, cholesterol, maximum heart rate, and chest pain type. Comprehensive exploratory data analysis (EDA) revealed significant correlations, such as the relationship between chest pain type and heart disease. Visualizations using Tableau provided insights into feature distributions and demographic patterns. Machine learning models like Logistic Regression, Support Vector Machine (SVM), and Random Forest were implemented after preprocessing steps like data normalization. Random Forest emerged as the most accurate model with 80% accuracy, evaluated using metrics like precision, recall, F1-score, and confusion matrix. This project highlights the application of machine learning in healthcare analytics for risk assessment and decision support.

Stock Price Analysis and Forecasting with Web Scraping and Prophet

This project scrapes Tesla's historical stock data from Yahoo Finance using BeautifulSoup, organizing it into a structured dataset suitable for analysis. Visualizations created with Plotly Express highlight trends in stock prices and trading volumes over the past year. Using the Prophet library, a machine learning model is trained on the historical data to forecast stock prices for the next 30 days. The project also evaluates the accuracy of the forecast model by comparing predicted values with actual data, ensuring reliable insights. Although this project focuses on Tesla, the forecasting and visualization pipeline can be applied to any stock with available historical data. Key steps include web scraping, data cleaning, visualization, model training, and future price prediction, offering insights into stock performance and potential trends.

Layoffs Data Analysis using MySQL

The Layoffs Data Analysis project utilizes MySQL to clean and structure a layoffs dataset sourced from Kaggle. This dataset, containing key information such as company, location, industry, total layoffs, layoff percentage, date, stage, country, and funds raised, was imported into MySQL Workbench. I created three tables—layoffs for raw data, layoffs_stagging for intermediary data, and layoffs_stagging_2 for the final cleaned data. The cleaning operations involved handling missing values, formatting dates, and standardizing company and location names to ensure consistency. The final, cleaned data stored in layoffs_stagging_2 is ready for analysis and visualization, demonstrating my ability to prepare structured data for actionable insights.
